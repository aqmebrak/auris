## Codebase Patterns

- **Vitest setup**: Use a separate `vitest.config.ts` (import `defineConfig` from `vitest/config`), NOT the `vite.config.ts`. Mixing vitest config into vite config causes `pnpm check` to fail with "does not exist in type 'UserConfigExport'". Also install `happy-dom` as the test environment.
- **Unit test environment**: `environment: 'happy-dom'` in `vitest.config.ts` gives access to `localStorage` in tests; `localStorage.clear()` in `beforeEach` prevents state leakage.
- **`pnpm lint` vs direct prettier**: `pnpm lint` (the quality-check command) exits 0 even when `npx prettier --check .` reports issues in pre-existing files. Trust `pnpm lint` for quality gate.
- **`svelte/no-navigation-without-resolve` lint rule**: `svelte.configs.recommended` enforces this rule. Always use `import { resolve } from '$app/paths'` and `href={resolve('/')}` instead of bare `href="/"` in `.svelte` files to satisfy ESLint. The autofixer also flags this as "Unexpected href link without resolve()." This applies to shadcn-svelte installed components too — `button.svelte` needs `import { resolve } from '$app/paths'` and `href={disabled ? undefined : resolve(href)}`.
- **shadcn-svelte `button.svelte` fix**: After `pnpm dlx shadcn-svelte@latest add button`, edit `src/lib/components/ui/button/button.svelte` to add `import { resolve } from '$app/paths'` in the second `<script lang="ts">` block and change `href={disabled ? undefined : href}` to `href={disabled ? undefined : resolve(href)}` to satisfy the `svelte/no-navigation-without-resolve` ESLint rule.
- **`{#each}` key requirement**: svelte-autofixer requires a key expression in `{#each}` blocks — always use `{#each items as item (item.uniqueField)}` form.
