## Codebase Patterns

- **Vitest setup**: Use a separate `vitest.config.ts` (import `defineConfig` from `vitest/config`), NOT the `vite.config.ts`. Mixing vitest config into vite config causes `pnpm check` to fail with "does not exist in type 'UserConfigExport'". Also install `happy-dom` as the test environment.
- **Unit test environment**: `environment: 'happy-dom'` in `vitest.config.ts` gives access to `localStorage` in tests; `localStorage.clear()` in `beforeEach` prevents state leakage.
- **`pnpm lint` vs direct prettier**: `pnpm lint` (the quality-check command) exits 0 even when `npx prettier --check .` reports issues in pre-existing files. Trust `pnpm lint` for quality gate.
